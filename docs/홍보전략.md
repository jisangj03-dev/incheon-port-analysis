# 홍보 전략 v1 (2026-07-14)

## 목표 정의
이 프로젝트의 홍보 목표는 오디언스·조회수가 아니라 **취업 신호**다. 성공 지표는 채용 과정(서류·면접)에서 이 저장소 링크가 실제로 열리고 프로젝트가 대화의 소재가 되는 것이다. 대중 채널(유튜브 등)은 범위 밖이다.

## 3단 구조
- 1단계 — 도착지 정비: README 30초 테스트. 처음 방문한 채용담당자가 30초 안에 "무엇을·어떻게·누가 직접"을 파악하게 한다. [2026-07-14 완료]
- 2단계 — 유통 경로 = 취업 서류: 이력서 프로젝트 단락 + 포트폴리오 1장. (코워크 첫 실사용 후보)
- 3단계 — 공개 채널 1개 = 링크드인: 프로필 정비 + 프로젝트 등록 + 시리즈 1 완결 포스트 1건.

## 관찰 게이트
3단계 게시 후 4주 반응 관찰(프로필 조회·링크 유입·연결/문의). 반응 데이터가 열 때만 확장을 검토한다(과정 기록 콘텐츠 등). 데이터가 게이트를 연다.

## 하지 않는 것
- 대중 채널 선행, 과정 기록 콘텐츠 선행(보류), 광고 의존 모델(기각 유지).
- AI 활용 은폐 — README·보고서 명시 유지(정직 하한선).

## 확장 사다리 (2026-07-14 확정 — 게이트는 데이터가 연다)

장기 비전: 보고서가 쌓여 물류 데이터 분석 사이트로 성장하고, 오디언스 형성과 (검증 통과 시) 전자책 등 후속으로 이어지는 것. 비전은 유효하며, 각 단계는 아래 게이트를 통과할 때만 착수한다.

- **현재 — 오디언스 실험 1차.** 홍보 3단계(링크드인 게시)가 곧 "사람 모으기"의 첫 데이터 수집이다. 게시일에 측정 지표와 판정 기준을 선커밋하고 4주 관찰한다.
- **게이트 1 — 사이트 승격.** 보고서 6편 이상 / 상호참조·분류 필요 / PDF 수요 — 셋 중 하나 충족 시 허브를 사이트 형태로 개편한다(Quarto 재검토 조건과 동일, docs/벤치마킹.md).
- **게이트 2 — 채널 확장.** 4주 관찰에서 실반응(프로필 조회·링크 유입·문의) 확인 시 과정 기록 콘텐츠·추가 채널을 검토한다.
- **게이트 3 — 전자책.** 오디언스에서 지불 신호 확인 시 5필터+렌즈 검증 세션을 연다. 사전 진단: 무료 공개 보고서의 재포장은 "직접 지불자 존재"와 "비커모디티" 조건에서 걸린다. 무료로는 얻을 수 없는 것이 무엇인지 데이터로 규명되어야 문이 열린다.
- **판별식.** "이 행동이 보고서를 늘리는가, 또는 이미 있는 보고서를 채용자 눈에 가깝게 옮기는가?" 둘 다 아니면 게이트 앞에 세운다.
