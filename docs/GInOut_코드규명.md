# 공컨 API `GInOut`·`ocCt` 코드 규명

> 목적: 공컨테이너 화물 통계 API(`ipaEmpConCargoInfo`) 응답의 `GInOut`(1~4)·`ocCt`(1~2) 코드가 각각 무엇을 뜻하는지 **실측 데이터로 추정**하고 근거를 남긴다.
> 기록일: 2026-07-11 / 근거: 2025년 1~3월 원시 행(totalCount=16) 직접 호출.
> **상태: [잠정: 실측 근거, 2026-07-11, 월요일 032-890-8254 공식 확인 예정]**

## 1. 상태

**[잠정]** — 아래 판독은 실측 데이터(검산 일치·값 분포·필드명)에 근거한 추정이다. API 응답 자체에 코드 라벨이 없어, **인천항만공사(032-890-8254)에 월요일 전화로 공식 정의를 확인한 뒤 [확정]으로 승격**한다. 후속 분석(#03 방향 분해)은 [확정] 승격을 착수 조건으로 한다.

## 2. 실측 크로스탭 (2025년 1~3월, GInOut × ocCt)

원시 응답은 월마다 여러 행으로 나뉘며, 그 축이 `GInOut`과 `ocCt` 두 개였다. 1~3월 16행을 조합별로 집계하면:

| GInOut | ocCt | 행수 | forEmpTeu 합 | korEmpTeu 합 | 외국적 비중(%) |
|:------:|:----:|:----:|-------------:|-------------:|:-------------:|
| 1 | 1 | 3 | 34,354.0 | 890.0 | 97.47 |
| 2 | 1 | 3 | 164,530.5 | 26,251.0 | 86.24 |
| 3 | 1 | 3 | 2,228.0 | 0.0 | 100.00 |
| 4 | 1 | 3 | 1,113.0 | 7.0 | 99.38 |
| 1 | 2 | 2 | 0.0 | 0.0 | — |
| 2 | 2 | 2 | 0.0 | 0.0 | — |

- `ocCt` 고유값: **1(12행) · 2(4행)**, 총 2종. **`ocCt=2`는 전 행이 0**(TEU·규격별 세부 모두 0) → 실적은 전부 `ocCt=1`에 실림.
- `GInOut` 고유값: **1·2·3·4**. `ocCt=1` 기준 네 코드 모두 매월 존재(각 3행).

## 3. 판독 논리

세 갈래 근거로 코드 의미를 좁혔다.

### (가) 검산 일치 → 상호배타 → #01·#02 수치 유효
- 1~3월 원시 전체 합계 `forEmpTeu=202,225.50 / korEmpTeu=27,148.00` 이 `container_2025.csv`의 1~3월 합계와 **소수점까지 완전 일치**.
- 즉 GInOut×ocCt 조합은 **상호배타(중복 없이 전체를 분할)**. 어떤 코드가 무슨 뜻이든 **단순 합산이 곧 총량**이므로, 이 축을 월별로 합산해 만든 **#01·#02의 수치는 그대로 유효**하다. (코드 규명은 수치 정정이 아니라 *분해 축 해석*의 문제)

### (나) 전 코드 외국적 우세 → 연안 배제 → 외/내항 축은 `ocCt`
- GInOut 1·2·3·4 **모두 외국적 비중 86~100%** (한국적 우세 코드 없음).
- 만약 어떤 GInOut 코드가 **연안(내항)** 이라면 한국적 선사가 우세해야 하는데 그런 코드가 없다 → **연안은 GInOut 축에 없다.**
- 외/내항(외항선 vs 내항선) 구분은 대신 **`ocCt`가 담는 축**으로 판단. `ocCt=1`(외항, 실적 전량) · `ocCt=2`(내항, 공컨 실적 0)로 정합.

### (다) 필드명 · 물류 정합 · hwpx 서술 → 1=반입, 2=반출, 3·4=환적
- 필드명 `GInOut` = **Gate In/Out** → 1·2가 반입/반출 쌍.
- 물류 정합: `GInOut=2`가 압도적 최대(외국적 16.5만 TEU) → 공컨은 **반출(수출 대기 빈 컨 반출·재배치)** 물량이 크다는 흐름과 일치. `GInOut=1`(반입)이 그다음.
- 잔여 3·4는 소량(합 3천여 TEU)·한국적 거의 0 → **환적(T/S)** 계열로 판단. (hwpx 서술도 방향 축을 반입/반출·환적으로 언급)
- 결론(잠정): **1=반입 · 2=반출 · 3·4=환적**.

## 4. 후속 보고서 재료 (메모)

- **`ocCt`(외/내항 구분)** 및 **규격별 필드** `forEmp_10 / _20 / _40 / _99`, `korEmp_10/_20/_40/_99`(10·20·40ft·기타 컨테이너 규격 분해)는 이번 규명엔 안 썼지만, **후속 보고서(규격 구성·외내항 분해)의 재료**로 보관.
