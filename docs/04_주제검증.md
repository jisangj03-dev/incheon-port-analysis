## 1. 후보

| 후보 | 내용 | 데이터원 |
|---|---|---|
| ① 터미널별 | 신항 / 남항 / 국제여객부두 컨테이너 물동량 구성 | 인천지방해양수산청 hwpx 월별 공표자료 |
| ② 연도 비교 | 2024 vs 2025 공컨테이너 | 공공데이터포털 공컨 API (15157693) |
| ③ 규격별 | 10 / 20 / 40 / 99 규격 구성 | 기존 수집 CSV (2025 공컨) |

### 사전 정정 — "신항" ≠ "SNCT"
인천항 컨테이너 전용부두는 남항(ICT·SICT·E1CT)과 인천신항(SNCT·HJIT)으로 나뉜다. hwpx의 "신항" 수치는 SNCT와 HJIT 두 운영사의 합이며 특정 운영사 단독 실적이 아니다. 본 시리즈는 "신항"을 특정 기업의 실적으로 서술하지 않는다. 운영사별 실적은 별도 데이터원을 확보한 경우에만 다룬다.

## 2. 선커밋 판정 기준 (결과 기입 전 확정)

### 선정 로직
- 후보 ①이 V1~V4를 모두 통과하면 #04 주제로 확정한다.
- 후보 ②·③은 결과와 무관하게 검증 결과를 기록하고, 통과 시 #05 이후 후보로 이월한다.
- ①이 실패하는 경우에만 ②·③ 중 검증 결과가 나은 쪽을 채택한다.

### 후보 ① 검증 항목
- V1 파일 존재: 2025년 1~12월 hwpx 12개가 모두 확보되어 있는가.
- V2 구조 일관성: 12개 파일에서 터미널 표의 행 라벨(원문)·열 구성·단위가 일관되게 읽히는가.
- V3 추출: 12개월 × 부두별 TEU 매트릭스가 추출되는가.
- V4 검산: 부두별 합계가 #02에서 발행에 사용한 월별 전체 컨테이너 총계와 일치하는가.
- V5 (보너스): 전년동월·증감 열이 존재하는가. 존재 시 후보 ②의 분모 확보로 연결된다.

판정: V1~V4 전부 통과 → 채택. 파일 결측이 있으나 보충 가능 → 채택 유지. 표 구조가 월마다 달라 일반화 불가하거나 검산 실패(정체불명 갭 1% 초과) → 기각.

### 후보 ② 검증 항목
- 2024년 1~12월 단일 조회로 12개월이 완결되는가.
- 필드 스키마와 축(GInOut 4코드, ocCt)이 2025와 동일한가.
- V5 성립 시, 2024 공컨 비율이 20~40% 대역에 들어오는가.

판정: 완결 + 동일 스키마 → 뱅크. 결측 또는 스키마 상이 → 사실을 기록하고 탈락.

### 후보 ③ 검증 항목
- (a) 문서: 규격 필드(_10/_20/_40/_99)의 정의가 공식 문서로 확인되는가.
- (b) 지문: 기존 CSV에서 규격 필드의 합이 TEU 필드와 어떤 관계인지 검산으로 확인되는가.
- (c) 서사: 방향별·월별로 규격 구성에 유의미한 차이가 있는가.

판정: (a)+(b) 이중 확정 + (c) 성립 → 뱅크. 이중 확정 실패 → 기각(정의가 미확정인 필드로는 보고서를 쓰지 않는다). 정의는 확정됐으나 구성이 단조 → 단독 보고서가 아닌 부록 소재로 강등.

### 용어 규칙
1개년 데이터만으로는 "계절성"이라는 표현을 쓰지 않는다. "월별 패턴"으로 적는다.
