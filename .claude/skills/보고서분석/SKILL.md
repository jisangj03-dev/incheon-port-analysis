# 보고서 분석 세션 SKILL

> **이 스킬은 「분석 세션」 전용이다.** 새 보고서의 데이터를 수집·검증·집계하고 차트를 만들어 **로컬 커밋**까지만 한다.
> **발행(README 등재·전편 링크·push)은 절대 이 세션에서 하지 않는다** — 그건 `보고서발행` 스킬이 담당한다. (세션 분리 원칙: 분석과 발행은 한 세션에 섞지 않는다. 비용·집중도 관리.)

## 이 스킬이 담는 것 / 담지 않는 것

- **담는 것(기계적 반복):** 폴더 점검, 원시 수집 규칙, 검증 게이트 체크리스트, 집계·차트·표 규칙, 로컬 커밋.
- **담지 않는 것(판단):** 주제 선정, 데이터 소스 적합성 판정, 코드 정의 확정, 해석 문장 작성, 발행 승인. → 이들은 모두 **⛔ 정지선**에서 회장님이 결정한다. 스킬은 정지선을 넘어 자동 진행하지 않는다.

---

## 1. 시작 점검

1. **작업 폴더 확인**: `문서\역량부서\인천항물동량` 인지 반드시 먼저 확인한다.
   - ⚠️ `수출서류랩`(다른 프로젝트)과 절대 섞지 않는다. 엉뚱한 폴더면 파일이 뒤섞인다.
2. **기준 문서 읽기**: `docs/다음세션_인수인계.md`(현재 상태·§7 다음 작업), `docs/GInOut_코드규명.md`(코드 정의 [확정] 여부), 해당 주제의 `docs/NN_주제검증.md`가 있으면 함께 읽는다.

## 2. ⛔ 정지선 — 착수 전제 확인

**주제와 데이터 소스 적합성이 이미 [확정]된 상태에서만 착수한다.**

- 분석 축(예: `GInOut` 같은 코드)의 정의가 `docs/GInOut_코드규명.md`에서 **[확정]** 인지 확인한다. **[잠정]** 이거나 미규명이면 **여기서 멈추고 회장님 확인.**
- 주제가 인수인계 §7/주제검증에서 확정되지 않았으면 **여기서 멈추고 회장님 확인.** 스킬이 임의로 주제를 정하지 않는다.

## 3. 데이터 수집 (원시 그대로)

- **엔드포인트**: `https://apis.data.go.kr/B551504/ipaEmpConCargoInfo/getEmpConCargoInfo`
- **파라미터**: `serviceKey`, `searchYear`(예: `"2025"`), `searchStartM`(`"01"`), `searchEndM`(`"12"`), `numOfRows`(`"100"` — 2025년은 64행이라 한 페이지에 전부), `pageNo`(`"1"`)
- **인증키**: 코드에 키를 직접 쓰지 않는다. `from config import SERVICE_KEY` 로 불러온다.
  - `config.py`는 `.gitignore`로 제외되어 저장소에 올라가지 않는다. 새로 받으면 `analysis/config_example.py`를 복사해 `analysis/config.py`로 만들고 키를 넣는다. **절대 커밋하지 않는다.**
- **XML 파싱**: 원본 바이트로 파싱한다 — `ET.fromstring(resp.content)` (한글 안전). 문서 기록이 아니라 **실제 응답 XML의 태그명**을 직접 확인한다(`items[0]`의 child 태그 출력).
- **원시 CSV 저장 규칙 (가공 금지)**:
  - 응답을 **가공 없이 원시 그대로** DataFrame → CSV로 저장한다. `encoding="utf-8-sig"`.
  - **미사용 필드도 반드시 보존한다.** 핵심 6태그(`yyyy, mm, GInOut, ocCt, forEmpTeu, korEmpTeu`)에 더해 **규격별 8필드**(`forEmp_10/_20/_40/_99`, `korEmp_10/_20/_40/_99`)를 함께 추출·저장한다. 이번 분석에 안 써도 **후속 재료 + 재현성 보험**으로 남긴다.
  - 검증·집계는 원시 CSV를 저장한 **뒤** 별도 숫자 사본(`df.copy()` 후 `pd.to_numeric`)에서만 한다. 원시 CSV는 형변환 없이 그대로 둔다.

## 4. 검증 게이트 (analyze_direction.py 기준)

**⚠️ 게이트가 하나라도 FAIL이면 `sys.exit(1)`로 즉시 종료한다. 이후 단계(집계·차트)로 절대 진행하지 않는다.** 허용오차 `TOL = 0.01`.

| 게이트 | 검사 내용 |
|--------|-----------|
| **G1 — 스키마·행수·코드값 집합** | 총 행수 == 64, `GInOut` 값집합 ⊆ {1,2,3,4}, `ocCt` 값집합 ⊆ {1,2} |
| **G2-① — 무효 축 필터** | `ocCt=2`(연안항) 전 행의 모든 수치 필드 == 0 (최대 \|값\| ≤ TOL). 공컨 실적은 전부 `ocCt=1`(수출입항)에 실림 |
| **G2-② — 기존 발행 수치와 월별 대조** | `ocCt=1` 월합(`forEmpTeu+korEmpTeu`)이 `container_2025.csv`(#01 산출물)의 `전체공컨TEU`와 **12개월 모두 일치** |
| **G2-③ — 연간 총합 대조** | `ocCt=1` 공컨 연간 총합 == 991,170 |
| **G3 — 기존 확정치와의 회귀 검산** | 1~3월 부분합(`ocCt=1`)의 `GInOut`별 합이 기존 확정치와 일치 (예: 수입 35,244.0 / 수출 190,781.5 / 수입환적 2,228.0 / 수출환적 1,120.0) |

- FAIL 처리는 공통 함수로: `[게이트] FAIL — 사유` 출력 후 "게이트 실패로 이후 단계를 진행하지 않고 종료합니다." → `sys.exit(1)`.
- 전 게이트 PASS 시에만 "✅ 모든 검증 게이트 PASS — 집계 단계로 진행합니다." 출력 후 집계로 넘어간다.

## 5. 집계·차트 규칙

- **집계 기준**: 유효 축만 사용한다(예: `ocCt=1`). 월별 × 분류 피벗, 결측 조합은 `fill_value=0.0`. 4분류(수입·수출·수입환적·수출환적)를 표시용으로 필요 시 3분류(수입·수출·환적=3+4)로 묶는다.
- **matplotlib 한글 폰트 (실제 코드 그대로)**:
  ```python
  plt.rcParams["font.family"] = "Malgun Gothic"   # 윈도우 기본 한글 폰트
  plt.rcParams["axes.unicode_minus"] = False       # 마이너스 기호 깨짐 방지
  ```
- **저장 경로·파일명**: `../reports/images/` 에 저장, `dpi=150`, `figsize=(10, 6)`. 파일명은 `<주제키>_chart_<연도>.png` / 구성비는 `<주제키>_share_<연도>.png` 형식(예: `direction_chart_2025.png`, `direction_share_2025.png`).
- **캡션에 생략·한계 명시**: 규모가 작아 표시를 생략한 계열이 있으면 캡션에 밝힌다. 실제 예: `<sub>환적은 연 0.8%로 규모가 작아 표시를 생략했다.</sub>`

## 6. 표 규칙

- TEU는 **정수 반올림**으로 표기한다.
- **반올림 오차 각주**를 표 아래에 단다. 실제 #03 각주 그대로:
  > 원자료는 0.25 TEU 단위 소수를 포함하며 표는 정수 반올림했다. 반올림으로 행·열 합이 표기 합계와 최대 2 TEU 차이 날 수 있다.
- 비율(%)은 데이터 정밀도 한계에 맞춰 표기한다(공표 반올림 분모를 쓰는 비율 보고서라면 소수 첫째자리까지만 — #02 사례).

## 7. ⛔ 정지선 — 해석은 이 세션에서 쓰지 않는다

**해석 문장(3장 내용)은 분석 세션에서 작성하지 않는다.** 검증 게이트 결과·헤드라인 수치·표·차트만 콘솔로 보고하고 **멈춘다.** 해석·한 줄 결론은 발행 세션에서 회장님 확인을 거쳐 확정한다.

## 8. 세션 마감 (로컬 커밋만)

- **push 금지.** README·#01·#02·타 docs 수정 금지(발행 세션에서 일괄).
- 분석 산출물(스크립트·원시 CSV·차트·보고서 초안)만 **로컬 커밋 1건**.
- 커밋 메시지 형식 (#03 초안 커밋 참고):
  ```
  보고서 #NN 초안: <주제 요약> (분석 코드·원시 데이터·차트 포함)
  ```
- 마감 후 **"로컬 커밋 완료, push 대기. 발행 세션에서 일괄"** 상태로 보고한다.
